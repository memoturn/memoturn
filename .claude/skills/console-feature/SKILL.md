# Add a console feature

The console (`apps/console`) is Vite + TanStack Router (file-based routes) + TanStack Query. It talks to the API through one thin typed client. This picks up where the `add-endpoint` skill / `endpoint-builder` agent leave off (they stop at `apps/console/src/lib/api.ts`).

## Layers

1. **API client method** — `apps/console/src/lib/api.ts`. Add a method to the exported `api` object using the `get<T>` / `post<T>` / `del<T>` helpers and `qs(...)` for query strings. **Types come from `@memoturn/contracts`** (re-exported at the top of the file as `export type * from "@memoturn/contracts"`) — never declare a new response interface; only request-shape helpers (like `TraceFilters`) live here. List endpoints return `{ data: T[] }`, so unwrap with `.then((r) => r.data)`:
   ```ts
   listWidgets: () => get<{ data: Widget[] }>(`/v1/widgets`).then((r) => r.data),
   createWidget: (body: {...}) => post<Widget>(`/v1/widgets`, body),
   deleteWidget: (id: string) => del<{ deleted: boolean }>(`/v1/widgets/${encodeURIComponent(id)}`),
   ```

2. **Route** — `apps/console/src/routes/`. File-based: a file (or `dir/index.tsx`) exports `Route = createFileRoute("/path")({ component })`. Parametrized routes use a subdirectory with a `$param.tsx` file (see `routes/traces/$id.tsx`, `routes/datasets/`). **Never edit `routeTree.gen.ts`** — it's generated by `tsr generate` (runs in build/typecheck), and a hook blocks edits to it. Just add/rename the route file and the tree regenerates.

3. **Data fetching** — TanStack Query inside the component:
   ```ts
   const qc = useQueryClient();
   const { data } = useQuery({ queryKey: ["widgets"], queryFn: () => api.listWidgets() });
   const create = useMutation({
     mutationFn: () => api.createWidget(body),
     onSuccess: () => qc.invalidateQueries({ queryKey: ["widgets"] }),
   });
   ```
   Use a stable `queryKey` array (include params: `["dataset", name]`); invalidate the matching key in a mutation's `onSuccess` so the list refetches.

## Verify

- `bun run typecheck` — the route tree is generated and type-checked here; client types must line up with the contract types.
- Run the app (`bun run dev`, console on :3000) and exercise the page; the Vite dev proxy forwards `/api` → the API (:3001) with the session cookie. The active project is sent via the `x-memoturn-project` header (project switcher) automatically by the client.

## Notes

- The console never hand-maintains response types — that's the contract drift guard. If you need a new shape, add it to `packages/contracts/src/index.ts` first (see `add-endpoint`).
- Mutating endpoints are role-gated server-side (`VIEWER` gets `403`); surface that in the UI (disable controls / show the error) rather than assuming success.
